# 목업 이미지→객체 가져오기 기대 결과 (image_to_object_mock_v1)

이 문서는 `docs/qa-fixtures/image_to_object_mock_v1.json`을 **고급 기능 → "이미지 → 객체 (목업)"**
로 가져왔을 때의 기대 동작/결과를 정의한다. 이 기능은 **로컬 JSON만** 읽으며,
외부 전송·API 호출·이미지 픽셀 분석을 **하지 않는다**(Phase 1, 목업 전용).

## 좌표 매핑 규칙

- 소스 좌표계: `source` = 1200 × 800 px (좌상단 원점).
- 아트보드: 기본 90 × 60 mm, 월드 원점이 아트보드 중심.
- 종횡비를 유지한 채 아트보드의 약 90%에 맞춰 축소하고, 결과를 **아트보드 중심에 배치**한다.
  - 적용 배율 `scale = min(90·0.9/1200, 60·0.9/800) = 0.0675` (mm/px).
  - 매핑: `worldX = -40.5 + px·0.0675`, `worldY = -27 + py·0.0675`.
- `strokeWidth`, `fontSize`, `labelSize`, 각도호 `radius`는 월드(mm) 값으로 간주.
  - 단, 위치/크기 기하(점, 박스 w/h, 호 반지름)는 `scale`로 변환된다.

## 가져오기 요약 (기대값)

- **가져올 객체 10개 · 건너뛴 객체 3개**
- 종류별: line 2 · ellipse 2 · rect 1 · triangle 1 · polyline 1 · text 1 · labeler 1 · anglearc 1
  - (참고: `arrow`는 앱에 별도 타입이 없어 `arrowHead:"end"`인 **line**으로 변환된다.
     `circle`도 별도 타입이 없어 **정원 비율 ellipse**(w=h)로 변환된다.)
- 경고 4건:
  - 입력 경고 1건(작성자 제공): "예시용 목업 데이터입니다 …"
  - `#10` 지원하지 않는 type `"spline"` → 건너뜀
  - `#11` `line`: p2 누락 → 건너뜀
  - `#12` `text`: text 문자열 누락 → 건너뜀

## 시각적 기대 모습

- 아트보드 중앙을 기준으로 위 객체들이 흩어져 배치된다(좌상단 두 선, 우상단 사각형,
  좌하단 원/꺾은선/각도호, 중앙 타원, 우하단 삼각형).
- 삽입 직후:
  - 삽입된 10개가 **모두 선택된 상태**(selectedIds)로 들어온다.
  - 활성 도구가 **선택(V)** 으로 돌아간다.
  - **되돌리기(Ctrl+Z) 1회**로 10개가 한 번에 사라진다(그룹 1스텝 히스토리).
- 각 객체는 일반 객체와 동일하게 **선택·이동·편집·저장·내보내기** 가능해야 한다.
  - 텍스트/라벨러는 더블클릭 편집, 각도호는 라벨 편집이 정상 동작.

## 실패/안전성 기대

- `version`이 `image-to-object-v1`가 아니거나, `source`/`objects`가 없으면
  **삽입 버튼이 비활성**이고 상태줄에 사유가 표시된다(앱 상태 훼손 없음).
- 잘못된 JSON(구문 오류)도 경고만 표시하고 크래시하지 않는다.
- 지원하지 않는/필수 필드 누락 객체는 **건너뛰고** 요약의 경고 목록에 표시된다.
