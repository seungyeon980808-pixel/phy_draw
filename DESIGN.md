## 1. 아키텍처 결정 (되돌리기 비쌈 — 확정, 변경 금지)

### 1-1. 상태 모델 = 데이터가 진실 (data-as-truth)

**결론**: 모든 오브젝트는 JS 데이터 객체로 존재하고, SVG 화면은 그 데이터의 투영(projection)이다. **데이터를 바꾸면 화면이 따라 그려진다.**

```
{ id, type, ...속성 }  ← 진실
        ↓ 렌더
      SVG 화면         ← 보여주기만
```

**근거**:
- 이 도구는 "그리기"가 아니라 "편집"이 핵심이다. 선택·이동·회전·속성변경·Undo·저장·재편집이 모두 "오브젝트를 데이터로 다루기"를 전제한다.
- 저장 = 배열을 JSON.stringify (공짜). Undo = 이전 상태 스냅샷. 인스펙터 = 데이터 한 줄 수정 → 화면 자동 갱신. 레이어 z순서 = 배열 순서.
- 대안(SVG DOM이 진실)은 시작 30분이 빠르지만, DOM을 거꾸로 읽는 코드가 늘면서 PyQt에서 겪은 불편(객체=화면이라 생기는 동기화 부담)을 웹에서 재현하게 된다.

**PyQt 대비**: 원본의 `QGraphicsItem`은 "객체=화면"이라 `it.scene() is not None` 체크가 사방에 깔려 있었다. data-as-truth에선 이 방어 코드가 원천적으로 불필요하다 — 삭제는 배열에서 빼면 끝, z순서는 배열 순서가 곧 답.

### 1-2. 좌표계 = SVG viewBox

**결론**: 줌·팬은 SVG의 `viewBox`로 구현한다. CSS transform 안 씀.

**근거**:
- data-as-truth와 궁합: 오브젝트의 x,y는 항상 월드 좌표 그대로, viewBox는 보여주기만 한다.
- 클릭 판정(히트테스트)이 쉽다. CSS transform은 역행렬을 직접 풀어야 한다.
- **고정 크기 선택 핸들**(아래 5-2)을 `핸들크기 / zoom` 한 줄로 구현 가능.

### 1-3. 동기화 = Vanilla JS + 얇은 store

**결론**: 프레임워크(React 등) 안 씀. 대신 30~40줄짜리 작은 store 패턴으로 "데이터 바뀜 → 화면 다시 그림"을 처리한다.

```js
state.subscribe(render)   // 데이터 바뀌면 render 자동 호출
state.update(...)         // 바꿀 땐 항상 이걸로
```

**근거**:
- 캔버스 에디터는 60fps 드래그·실시간 핸들 등 "프레임워크가 오히려 방해되는" 인터랙션이 많다. React의 리렌더 모델과 마찰.
- 빌드 도구 불필요 → GitHub Pages에 파일 그대로 올리면 끝(의존성 최소화 원칙).
- store 패턴(구독→갱신)은 모든 상태관리의 공통 원리라, 작게 직접 만들면 학습에도 좋다.
- 순수 Vanilla보다 나은 점: data-as-truth라 동기화가 반드시 깔끔해야 하는데, 수동 호출은 빠뜨리기 쉽다. 얇은 store가 그 실수를 구조적으로 막는다.

## 2. 데이터 스키마 (되돌리기 비쌈 — 확정)

PyQt 원본 `items.py`의 두 갈래 기반 클래스를 그대로 옮긴다.

### 2-1. 두 갈래 구조

**갈래 A — 크기 기반** (사각형·타원·삼각형 등): 위치 + W/H + 회전으로 정의. 원본 `BasePhysicsItem` 계열.

**갈래 B — 끝점 기반** (선·꺾은선): 끝점들로 정의. W/H 개념 없음. 원본 `TwoPointItem` 계열.

```
공통 필드:
{ id, type, rotation, strokeLevel, strokeWidth,
  fillLevel, fillNone, locked, layerId, order }

갈래 A 추가: { x, y, w, h }
갈래 B 추가: { points:[{x,y}...], arrowHead, dashStyle, dashGap, dashLength }
```

**근거**: 두 갈래는 크기조절·회전 규칙이 원래부터 다르다(원본 `ResizeItemCommand`가 'size'와 'p1/p2'를 분기 처리한 이유). 이건 원본이 이미 검증한 분기다.

### 2-2. 속성 필드 의미 (인스펙터와 1:1 대응)

- `strokeLevel`: 선 명도, 0 = 검정 (무채색)
- `strokeWidth`: 선 굵기 (pt). 기본값은 기본설정에서 세팅
- `fillLevel`: 채우기 명도 (무채색)
- `fillNone`: 채우기 없음 토글 (true여도 클릭은 됨 — 5-3 참조)
- `locked`: 개체 보호 (6-3)
- `layerId`: 소속 레이어 (1/2/3)
- `order`: 같은 레이어 내 z순서

## 3. 도구 (8종)

| 단축키 | 도구 | 분류 |
|---|---|---|
| `V` | 선택 | 다루기 |
| `L` | 직선 | 그리기 |
| `P` | 꺾은선(펜) | 그리기 |
| `R` | 사각형 | 그리기 |
| `O` | 타원 | 그리기 |
| `Y` | 직각삼각형 | 그리기 |
| `C` | 곡선(2점 호) | 그리기 |
| `T` | 텍스트 | 그리기 |

- **선택도구(V)**: 모든 편집의 출발점. 이동·크기조절·회전. 도형을 그린 직후 자동으로 이 도구로 복귀.
- 단축키 글자 자체는 되돌리기 싼 결정 — 구현하며 조정 가능.

### 3-1. 선 (L / P) — 이 도구의 主무기

물리 그림의 9할이 선이다. 그만큼 옵션이 풍부하다. (속성으로 처리 — 별도 도형 안 만듦)

- **L (직선)**: 왼클릭 시작 → 왼클릭 끝. 점 2개로 완료.
- **P (꺾은선)**: 왼클릭마다 점 추가(계속 꺾임) → **더블클릭 또는 Enter로 종료**.
  - 우클릭은 쓰지 않음(컨텍스트 메뉴 보존, 브라우저 충돌 회피). "끝내기가 특별 신호, 꺾기가 기본"인 펜 도구 표준.
  - 그리던 중 다른 도구 단축키를 누르면 미완성 선은 자동 폐기.
- **공통 속성**:
  - 실선 / 점선 (점선은 간격·길이 조정 가능 — SVG `stroke-dasharray`)
  - 화살표 위치: 없음 / 끝 / 중앙 / 양끝 — **중앙이 물리 광선용으로 중요**(방향만 표시, 선은 이어짐)
  - 화살표 머리 크기 = **선 굵기에 연동**(굵기 × 배수). 가는 선에 큰 머리가 붙는 불균형 방지. 배수 조정은 인스펙터에서 열어둘 수 있음.
  - Shift = 15도 스냅

### 3-2. 닫힌 도형 (R / O / Y)

- **R 사각형**: 모서리 둥글기 없음(직각만). 정사각형은 인스펙터 토글.
- **O 타원**: `{rx, ry}` 구조, 정원은 rx=ry 특수형으로 흡수(별도 원 도구 없음). 정원은 인스펙터 토글.
- **Y 직각삼각형**: 빗면(역학)·프리즘(광학)용. 자유삼각형이 필요하면 P(꺾은선)로 점 3개.
- 모두 기본 **테두리만, 속 투명**. 채움은 옵션.

### 3-3. 곡선 (C)

- **2점 호(arc)만**: 시작·끝점 + 휘는 정도. 베지어 자유곡선은 v1 제외(제어점 편집 UI가 무거움).
- 물리 그림의 곡선은 대부분 "살짝 휜 선" 수준이라 호로 충분.

### 3-4. 텍스트 (T)

- 라벨용("물체", "볼록렌즈", "상").
- 폰트 = IBM Plex Sans KR 1개 고정 + 크기 조절. (폰트 선택은 나중)
- **주의(원본 버그 회피)**: 텍스트 편집 중 backspace로 빈 내용이 돼도 박스가 통째로 삭제되면 안 됨. 편집 중 backspace와 도형 삭제(Delete)를 구분할 것.

## 4. 인터랙션 / 단축키

### 4-1. 변형 보조키 (확정)

| 키 | 동작 |
|---|---|
| **Shift** | 크기조절 시 **현재 비율 유지** (개별 도형). 선에선 15도 스냅 |
| **Ctrl** | 회전 시 **15도 단위 스냅** |
| 정사각/정원 | **인스펙터 토글** (Shift 아님) |

- 정사각형/정원 만들기를 Shift가 아니라 토글로 뺀 이유: Shift는 "현재 비율 유지"로 쓴다. "이미 만든 모양의 비율을 지키며 확대"가 일러의 1:1 강제보다 실용적.

### 4-2. 편집 단축키

| 단축키 | 동작 |
|---|---|
| Delete | 선택 개체 삭제 |
| Ctrl+C / Ctrl+V | 복사 / 붙여넣기 — **붙여넣기는 마우스 커서 위치에 생성** |
| 화살표 키 | 선택 개체 한 칸씩 이동 (미세조정 — 곡선 스냅 대체 수단) |
| PageUp / PageDown | **같은 레이어 안에서** 선택 개체 z순서 한 칸 이동(앞/뒤). 레이어 경계는 안 넘음 |

- Ctrl+D(복제) 없음. Esc 없음(V로 선택도구 가면 됨, 미완성 선은 다른 도구 누르면 자동 폐기).

### 4-3. 도형 생성 직후

- 그린 직후 **선택도구(V)로 자동 복귀** + 방금 그린 것이 선택된 상태. 바로 다듬거나 단축키로 다음 도구.

## 5. 선택 / 히트테스트 (되돌리기 비쌈 — 확정)

### 5-1. 클릭 판정 = 도형의 실제 모양 기준

**결론**: 클릭해서 잡히는 범위(히트 영역)는 **도형의 실제 모양 그대로**다. 사각 바운딩 박스가 아니다.

**근거**: 삼각형의 바운딩 박스는 사각형이라, 삼각형 바깥의 빈 구석(박스 안이지만 삼각형 밖)을 클릭하면 삼각형이 잡히는 오클릭이 생긴다. 이게 큰 스트레스 요인. 실제 모양 기준이면 구석은 안 잡힌다.

**구현**: SVG는 도형을 실제 모양(`<polygon>` 등)으로 그리면 클릭 판정이 그 모양대로 된다(PyQt의 `shape()`를 SVG가 대신 해줌). 비스듬히 기운 도형도 실제 외곽 기준이라 정확.

### 5-2. 바운딩 박스 vs 히트 영역 (분리 개념)

| | 모양 | 역할 |
|---|---|---|
| 바운딩 박스 (파란 점선) | **사각형** | 보여주기 + 핸들 8개 자리 |
| 히트 영역 | **도형 실제 모양** | 클릭 판정 |

- 바운딩 박스는 사각형일 수밖에 없다(핸들이 모서리·변에 붙어야 함). 이건 시각 표시일 뿐.
- **선택 핸들은 고정 크기**: 줌 배율과 무관하게 항상 같은 픽셀 크기(`핸들크기 / zoom`). 오브젝트가 작아도 핸들은 동일.

### 5-3. 빈 도형도 선택 가능

**결론**: `fillNone`(채우기 없음)이어도 내부 클릭으로 선택된다. "색이 없다"와 "클릭이 통과한다"를 분리한다.

**구현**: 빈 도형에 `fill="transparent"`(투명 채움)를 깐다 — 눈엔 안 보이지만 클릭은 받아낸다. 채우기 상태:

| 인스펙터 설정 | 보이는 모습 | 클릭 |
|---|---|---|
| 채우기 없음 (fillNone) | 속 비어 보임 | **잡힘** (투명 채움이 받음) |
| 채우기 명도 지정 | 그 명도로 채워짐 | 잡힘 |

### 5-4. 기타 (표준값)

- **얇은 선 클릭**: 1px 선은 정확히 클릭 불가 → 클릭 허용 범위 약간의 여유(몇 px) 추가.
- **다중 선택**: 드래그 박스 + Shift+클릭.

## 6. 오브젝트 관계 (스냅·그룹·보호·레이어)

> 핵심: 6-2(그룹 멤버), 6-3(보호), 6-4(비활성 레이어)는 모두 **"선택은 되는데 수정은 막는"** 같은 메커니즘의 변주다. 한 번 만들어 공유한다.

### 6-1. 스냅 (자동 부착)

**결론**:
- 오브젝트를 끌어 다른 오브젝트에 가까이 가면 → 붙을 지점에 **빨간 점** 예고 → 더 가까이/놓으면 **딱 부착**.
- 부착 기준 = **실제 도형 외곽**(바운딩 박스 아님 — 5-1과 같은 이유. 비스듬한 도형은 박스로 하면 안 닿았는데 붙는 문제).
- **대칭 도형은 대칭축까지 정렬**됨(예: 렌즈를 광축에 — 좌우대칭축 기준 정렬). 대칭 정렬은 스냅과 한 몸.
- **v1 범위**: 직선 외곽 도형(사각·삼각·기운 것)부터 적용. **곡선 접점 스냅(원이 직선에 접하는 등)은 나중** — 빈도 낮고, 키보드 화살표 미세조정으로 대체 가능.

**근거**: 곡선의 접점 계산은 무겁다(곡선 위 무한한 점 중 접점을 풀어야 함). 자주 쓰는 "렌즈를 광축에"는 곡선 접점이 아니라 대칭축 정렬로 해결되므로, 진짜 곡선 접점은 미뤄도 실사용에 지장 없음.

**격자 스냅**: 격자는 기본 보기용. 단 **직선이 격자선과 평행한 경우에만** 격자 스냅 허용. 곡선·사선은 격자 스냅 안 함(화살표로 미세조정).

### 6-2. 그룹 (묶기 / 풀기)

원본 `GroupCommand` / `PhysicsGroupItem` 재현. 단 표시 방식은 아래처럼 **변경**한다.

- **묶기**: 여러 도형 선택(Shift+클릭 또는 드래그) → 묶기. 원본 `PhysicsGroupItem.fromItems()`.
- **선택·변형**: 그룹 클릭 시 **전체가 한 덩어리**로 선택. 같이 이동·회전·크기조절.
- **크기조절 = 항상 비율 고정** (Shift 무관, 강제). 근거: 묶였다는 건 도형 간 상대 배치가 의미를 갖는다는 뜻 — 비율을 깨면 광선 각도·간격 등 관계가 망가진다. 회전·이동은 자유(상대 배치 안 깨짐).
- **묶인 표시 (원본과 다른 점)**: 원본은 그룹 전체를 감싸는 큰 박스 하나였으나 → **각 개체마다 따로따로 초록색 바운더리**로 표시. 어떤 도형들이 묶였는지 한눈에 보이게. (파란=개별 선택, 초록=그룹)
- **개별 지목 → 풀기**: 그룹 안 도형을 **더블클릭**하면 그 하나가 지목됨. 지목 상태에선 **수정 전부 불가, 우측 패널 "개체 풀기"만 가능**. (Shift는 안 씀 — 다중선택·비율유지와 충돌하므로. 더블클릭은 일러 표준)
  - 풀기 = 그룹 전체 해제(다시 다 개별로). 부분 해제(멤버 한 명만 빼기)는 안 함.

### 6-3. 개체 보호 (잠금)

원본 `LockCommand` / 인스펙터 "위치 고정" 재현.

- 켜면: **선택은 되지만 모든 수정 불가 + 캔버스에 고정**. 풀려면 우측 패널에서 보호 해제.
- 6-2의 "그룹 멤버 지목 상태"와 같은 상태(선택 O, 수정 X) — 메커니즘 공유.

### 6-4. 레이어 (고정 3개)

- **고정 3개** (레이어 1/2/3, 추가·삭제 없음, 이름변경만 가능). 무거워지지 않게 제한.
  - 추가·삭제 버튼이 없어 UI도 단순. 물리 그림엔 "배경/주요도형/라벨" 3층이면 충분.
- **순서(관행)**: 레이어 1(맨 아래/뒤) → 2 → 3(맨 위/앞). "위 행이 앞에 그려짐"(원본 인스펙터와 동일).
- **활성 레이어 = 레이어 패널에서 클릭한 레이어**.
- **비활성 레이어 = 선택·수정 불가 + 살짝 투명**(클릭 통과). 지금 작업 층에 집중. (이것은 6-3 보호보다 한 단계 강함 — 클릭 자체가 통과)

**상태 정리표**:

| 상태 | 선택 | 수정 | 모습 |
|---|---|---|---|
| 일반 개체 | O | O | 정상 |
| 보호된 개체 (활성 레이어) | O | X | 정상 |
| 그룹 멤버 (더블클릭 전) | 그룹째 O | X (풀기만) | 초록 테두리 |
| 비활성 레이어 개체 | **X** | X | 살짝 투명 |

## 7. 화면 구조 (3패널)

```
┌──────────────┬───────────────────────┬──────────────────┐
│ 기본 도구     │                       │   인스펙터        │
│ V L P R O     │                       │  (선택 오브젝트   │
│ Y C T         │       캔버스          │   속성 — 선옵션   │
│ ─────────     │    (SVG, viewBox)     │   포함, 접이식)   │
│ 템플릿        │                       │                  │
│ ▸ 광학        │   100mm 아트보드       │ ─────────────    │
│ ▸ 역학        │   고정핸들 (n/zoom)    │  AI 채팅 ◀토글▶  │
│ ▸ 파동        │                       │  (우측 하단)      │
│ ▸ 전기        │                       │                  │
└──────────────┴───────────────────────┴──────────────────┘
        [상단: 설정 버튼 — 기본 선굵기·단위·격자]
```

- **좌측 — 위아래 분할**: 위 = 기본 도구 8개(고정), 아래 = 템플릿 라이브러리(카테고리별 **접이식**). 사용 패턴이 "기본+광학", "기본+역학"이라 그날 단원 카테고리만 펴고 나머지 접음.
- **중앙 — 캔버스**: SVG + viewBox. 100mm 아트보드 경계 표시(8장 참조).
- **우측 — 인스펙터**: 모든 속성(선 옵션 포함)을 여기서. 상단 옵션 바 안 만듦(같은 속성이 두 군데 있으면 헷갈림). 세로로 길어지지 않게 **접이식 섹션**. 새 선 그릴 때 기본값은 직전 선 설정을 이어받음.
- **우측 하단 — AI 채팅**: **토글로 켜고 끔**. 안 쓸 땐 접어서 인스펙터 공간 확보.
- **상단 — 설정 버튼**: 기본 선굵기·단위(mm/cm/inch)·격자 설정. 자주 안 만지니 평소 숨김.

### 7-1. 인스펙터 내용 (원본 재현 — 이미지 2)

- 모양: 선 명도(0=검정) / 채우기 없음 토글 / 채우기 명도 / 면 채우기 패턴 / 선 굵기
- 크기·회전: W / H / 회전
- 잠금·고정: 위치 고정(=개체 보호) / 최상단 고정
- 레이어(쌓임 순서): 맨 앞으로 / 앞으로 / 뒤로 (+ PageUp/Down 단축)
- 레이어 패널: 3개 고정, 가시성 토글·잠금·이름변경

### 7-2. 색 선택기 (원본 재현 — 이미지 1)

무채색 전용이라 1D로 충분(색상환 불필요).

- **그라데이션 바**: 흰→검 띠를 클릭/드래그로 명도 집기 (연속)
- **팔레트**: 미리 둔 회색 칸들, 클릭 선택·더블클릭 확정 (이산)
- **정밀 슬라이더**: 0~255 숫자 미세 조정
- **선택된 색** 미리보기
- 데이터엔 명도값으로 저장(`grayLevel` 0~255 또는 strokeLevel/fillLevel).

## 8. 출력 / 규격 / 미리보기

### 8-1. 작업 영역 = 최대 100mm × 100mm

- 시험지에 들어갈 그림의 **최대 크기**가 100mm × 100mm. 그 안에서 가로·세로 자유(예: 100×60 가능). 정사각 고정 아님.
- 캔버스 안에 **아트보드 경계**(100mm 범위)를 표시. 그 안이 실제 출력 영역.
- **경계 밖 = 작업 공간이지만 출력 제외**(일러 아트보드처럼). 내보낼 때 아트보드 영역만 빠져나감.

### 8-2. 단위

원본 `units.py` 재현(거의 복붙). 내부는 px, 보여줄 때만 변환.

- mm(기본) / cm / inch / px. 단위 전환 가능.
- 내보내기 dpi 환산만 점검(웹은 96dpi 가정이 브라우저마다 미묘 — 인쇄 300dpi 목표).

### 8-3. 격자

원본 사양 재현: 10mm 단위 격자 + 50mm마다 작은 숫자 + 진하기 조절 + **저장(출력) 시 제외**. 기본 보기용(스냅은 6-1 격자 스냅 규칙).

### 8-4. 미리보기 (별도 모달)

**목적**: 단순 크기 확인이 아니라 — **"시험지 칸에 넣었을 때 주변 글씨와 통일감이 맞는가"**를 본다. 글씨 크기·선 굵기가 본문과 어울리지 않아 튀는 경우가 많기 때문.

**구성**:
- 모달에 **"샘플 문제 틀"**(윗줄 문제 글씨 + 그림 칸 + 아랫줄 글씨)을 띄움. 글씨는 시험지 본문과 같은 폰트·크기.
- 그 칸에 내 그림을 **실제 비율·실제 크기로 앉혀서** 보여줌. 핸들·격자 제거.
- 선 굵기·글씨 통일감을 눈으로 확인.
- **그림 위치 드래그 없음** — 실제 배치는 한글에서. 미리보기는 "확인"만.
- **샘플 틀 글씨 = 고정 더미 텍스트** (사용자가 예시 문장 제공 → 박아둠. 입력 기능 없음). 크기 비교 기준선 역할.

## 10. 개발 Phase (제안)

> 토대(아키텍처)를 먼저, 그 위에 기능을 얹는 순서. 물리 도형은 기본 엔진 완성 후.

- **Phase 1 — 기본 캔버스 엔진**: SVG+viewBox, store, data-as-truth, 기본 도형 7종(L/P/R/O/Y/C/T), 선택/이동/크기조절/회전, 고정 핸들 + 실제모양 히트테스트, Undo/Redo. *직선 도형 스냅까지.*
- **Phase 2 — 레이어 & 인스펙터**: 레이어 3개(활성 전환·비활성 투명), 레이어 내 z순서, 인스펙터(접이식, 색 선택기), 개체 보호.
- **Phase 3 — 그룹 & 물리 템플릿**: 그룹(초록 표시·비율고정·더블클릭 풀기), 물리 도형 라이브러리(광학부터 — 렌즈 등), 대칭축 정렬, 곡선 접점 스냅 정교화.
- **Phase 4 — 저장 & 내보내기 & 미리보기**: JSON 저장/불러오기, PNG(300dpi)·SVG 내보내기, 100mm 아트보드 출력, 미리보기 모달(샘플 틀 합성).
- **Phase 5 — AI 연동**: Claude API 채팅(토글), SVG 생성 → 데이터 객체로 정규화 후 삽입. (AI 이미지=래스터 편집은 v2.)
